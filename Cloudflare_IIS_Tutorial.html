🌍 Share Your IIS Website Globally

Using Cloudflare Tunnel to bypass ISP blocks & Routers

No Port Forwarding. No ISP Limits. Just Your Code.

Normally, ISPs block your public IP from being a "host". Cloudflare Tunnels create a secure outbound bridge from your PC to the world, giving you a public .trycloudflare.com link for free.

1

Install Cloudflare Tunnel

First, we need the tool that builds the "bridge". Open PowerShell as Administrator and run:

winget install Cloudflare.cloudflared
2

Check Your IIS Site Port

Open IIS Manager and look at your "Bindings". Most sites use port 80 or 8080. Make sure your site is Started.

Pro Tip: If your site is ASP.NET Core, check the IP binding. If it's bound to a specific IP (like 192.168.0.8), we'll need that in the next step!
3

Launch the Tunnel

Now, run the command to start the tunnel. Replace the IP/Port with your IIS details:

cloudflared tunnel --url http://localhost:80 --http-host-header localhost

If your site uses a specific IP inside IIS, use this instead:

cloudflared tunnel --url http://192.168.x.x:80 --http-host-header 192.168.x.x
4

Get Your Public URL

Look at the PowerShell output. You will see a big box with a link that looks like this:

https://your-random-name.trycloudflare.com

You're live! Anyone with this link can now visit your local IIS website from anywhere in the world.

❓ Why am I seeing a 404 or 400 error?

  • 400 Invalid Hostname: This happens if IIS doesn't recognize the Cloudflare link. Always use the --http-host-header localhost flag to fix this.
  • 404 Not Found: Check if your IIS site is actually running and has an index.html or index.php file.
  • 502 Bad Gateway: This means the tunnel is working, but it can't find your IIS server. Check the Port number!
